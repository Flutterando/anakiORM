## Vaden + Anaki Example

A REST API example using [Vaden](https://pub.dev/packages/vaden) with [Anaki](../../packages/anaki_orm/) and PostgreSQL.

Demonstrates:
- **Database configuration** via `@Configuration` + `@Bean`
- **RowAdapter** plugged from `DSON` — one line: `RowAdapter(dson.fromJson, dson.toJson)`
- **Repository** using `AnakiQueryBuilder` with fluent API
- **Migrations** with `Migrator` in an `ApplicationRunner`
- **CRUD controller** with `@Param`, `@Body`, `@Api`

## Prerequisites

- Dart SDK ≥ 3.10
- Docker (for PostgreSQL)
- Native library built: `./scripts/build_native.sh postgres --local` (from monorepo root)

## Run

```bash
# 1. Start PostgreSQL
docker compose up -d

# 2. Generate Vaden code
dart run build_runner build

# 3. Run server
dart run bin/server.dart
```

Server starts at `http://localhost:8080`.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/users` | List all users |
| `GET` | `/users/:id` | Get user by ID |
| `GET` | `/users/active` | List active users |
| `POST` | `/users` | Create user |
| `PUT` | `/users/:id` | Update user |
| `DELETE` | `/users/:id` | Delete user |
| `GET` | `/docs` | Swagger UI |
| `GET` | `/hello/ping` | Health check |

## Example requests

```bash
# Create
curl -X POST http://localhost:8080/users \
  -H 'Content-Type: application/json' \
  -d '{"name": "Ana", "email": "ana@example.com", "active": true}'

# List
curl http://localhost:8080/users

# Get by ID
curl http://localhost:8080/users/1

# Update
curl -X PUT http://localhost:8080/users/1 \
  -H 'Content-Type: application/json' \
  -d '{"name": "Ana Maria", "email": "ana@example.com", "active": true}'

# Delete
curl -X DELETE http://localhost:8080/users/1
```

│   └── resources/                      # Storage + static files

│   ├── dto/

migrations/
└── 001_create_users.sql                # SQL migration
